## 1. 项目概述
+ **项目名称：** 家庭小食堂
+ **项目背景：** 解决家庭日常生活中“不知道吃什么”、“众口难调”的问题，让家里负责做饭的人（大厨）能提前规划食材，让其他家人有参与感。
+ **核心目标：** 提供一个轻量级的点餐、许愿和备餐管理工具。

## 2. 角色定义
系统仅包含两个核心角色：

+ **家人（食客端/C端）：** 家里的点餐者（伴侣、孩子等），负责浏览菜单、点菜、催单、评价。
+ **大厨（管理端/B端）：** 家里负责买菜做饭的人，负责发布今日菜单、接收订单、管理状态。

## 4. 核心业务流程图（文字描述版）
1. **大厨备餐：** 早上/前一天晚上，大厨打开小程序，根据冰箱食材，将拿手菜库里的几道菜设为“今日上架”。
2. **家人点餐：** 家人打开小程序，看到今日可选菜单，选择自己想吃的菜，提交订单（选择今晚吃）。
3. **大厨接单：** 大厨查看统计单，发现今天大家一共点了：2份红烧肉、1份炒青菜。大厨开始备菜。
4. **进度通知：** 饭菜做好后，大厨在小程序点击“开饭啦”，家人微信收到消息提示（“您的红烧肉已上桌，快来吃饭！”）。
5. **饭后互动：** 家人给今天的菜打个满分，大厨获得成就感。

## 5. 补充说明 (非功能性要求)
+ **授权登录：** 由于是家庭内部使用，不需要复杂的手机号验证码，直接微信一键授权登录即可。管理员（大厨）权限可以通过硬编码微信号，或者第一个注册的人默认成为大厨来实现。
+ **UI风格：** 建议采用温馨、可爱的卡通风格或暖色调，界面字体可以稍微放大，方便长辈使用。
+ **部署建议：** 强烈建议使用**微信小程序·云开发**。因为家庭使用流量极小，云开发自带的免费额度（云数据库、云存储）完全够用，**0服务器成本**，且无需购买域名。
